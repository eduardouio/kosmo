Procesamiento de datos en texto plano a formato JSON
Objetivo:
Convertir datos de entrada en texto plano sobre floricultura a una estructura JSON predefinida.

Formato y estructura del análisis
Estructura general de los datos: Cada línea de texto contiene información sobre cajas de flores y su contenido. El formato puede variar, pero los elementos clave son:

Cantidad: Número de cajas.
Modelo de caja: Identificado como HB o QB.
Variedad de la flor: Nombre de la variedad.
Medida(s): Longitud de los tallos, puede haber una o más medidas.
Cantidad de tallos totales: Número total de tallos en la caja.
Precio(s): Costo de la caja basado en las medidas.
Precio de Tallo en Caja: Costo mas alto de tallo en caja, si solo hay un precio, se asume que es el precio de todos los tallos.
Consideraciones especiales:

Ignorar cualquier línea que comience con uno de los siguientes nombres (o sus variantes simplificadas):
Kosmo Flowers SA
Floraroma SA
Spring Roses
Valent Roses
Flores De La Hacienda
Florifrut S.A.
Fairis Garden
Rosas Del Campo
Yamiteo Flowers
Santa Clara
Moonlight Flower
Ejemplo detallado de análisis:

Entrada: 1hb Explorer 40/50 x 250 0,40/0,50

Cantidad: 1
Modelo de caja: HB
Variedad: Explorer
Tallas: 40, 50
Total de tallos: 250
Precios: 0.40 (para 40), 0.50 (para 50)
Precio de Tallo en Caja: 0.50
Salida en JSON A RESPETAR
[
    {
        "quantity_box": 1,
        "box_model": "HB",
        "tot_stem_flower": 250,
        "stem_cost_price_box": 0.50,
        "box_items": [
            {
                "variety": "Explorer",
                "tot_stem_flower": 0,
                "length": 40,
                "stem_cost_price": 0.40
            },
            {
                "variety": "Explorer",
                "tot_stem_flower": 0,
                "length": 50,
                "stem_cost_price": 0.50
            }
        ]
    }
]

Notas sobre el JSON esperado:

Cada línea procesada debe generar un objeto JSON dentro de una lista.
En caso de múltiples tallas o precios, se deben crear objetos separados dentro de box_items.
Si no se especifica un precio, se usará 0 como valor predeterminado.

Detalles técnicos adicionales:
Todas las longitudes y precios deben convertirse a números flotantes para precisión.
Los nombres de variedades deben estar en mayúsculas.
Los precios separados por / o espacios deben asignarse a sus respectivas tallas.

Las respuestas no deben estar con
[{'flowers': [{},{}]}]
siempre debe ser una lista de objetos [{},{},{}]
